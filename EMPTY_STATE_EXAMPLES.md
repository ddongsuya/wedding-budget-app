## 일러스트레이션 상세

### Wedding (결혼/웨딩)
- **색상**: #FFF1F2 (배경), #FDA4AF (하트), #E11D48 (리본)
- **요소**: 하트, 리본, 반짝이
- **느낌**: 로맨틱, 사랑스러움

### Budget (예산)
- **색상**: #FEF3C7 (배경), #FCD34D (지갑), #F59E0B (동전)
- **요소**: 지갑, 동전, 플러스 아이콘
- **느낌**: 금전적, 계획적

### Expense (지출)
- **색상**: #ECFDF5 (배경), #6EE7B7 (영수증), #10B981 (아이콘)
- **요소**: 영수증, 플러스 아이콘, 텍스트 라인
- **느낌**: 기록, 관리

### Venue (식장)
- **색상**: #FFF1F2 (배경), #FECDD3 (건물), #FDA4AF (지붕)
- **요소**: 웨딩홀 건물, 문, 하트
- **느낌**: 장소, 공간

### Checklist (체크리스트)
- **색상**: #EDE9FE (배경), #C4B5FD (클립보드), #8B5CF6 (체크)
- **요소**: 클립보드, 체크마크, 리스트 라인
- **느낌**: 할일, 완료

### Calendar (캘린더)
- **색상**: #DBEAFE (배경), #93C5FD (달력), #3B82F6 (헤더)
- **요소**: 달력, 날짜, 하트 표시
- **느낌**: 일정, 날짜

## 사용자 플로우

### 시나리오 1: 첫 사용자
1. 앱 설치 후 로그인
2. 대시보드 접속 → 데이터 없음
3. "지출" 탭 클릭 → **빈 상태 표시**
4. "첫 지출 기록하기" 버튼 클릭
5. 지출 추가 모달 열림
6. 지출 정보 입력 및 저장
7. 지출 목록에 첫 항목 표시

### 시나리오 2: 검색 실패
1. 지출 목록에서 검색
2. "웨딩케이크" 입력
3. 결과 없음 → **검색 결과 없음 표시**
4. "검색어 지우기" 클릭
5. 전체 목록으로 복귀

### 시나리오 3: 에러 복구
1. 네트워크 끊김
2. 데이터 로드 실패 → **에러 상태 표시**
3. "다시 시도" 버튼 클릭
4. 데이터 재요청
5. 성공 시 정상 화면 표시

## 테스트 방법

### 1. 지출 빈 상태 테스트
```typescript
// 브라우저 콘솔에서
localStorage.removeItem('expenses');
window.location.reload();
```

### 2. 식장 빈 상태 테스트
```typescript
// 브라우저 콘솔에서
localStorage.removeItem('venues');
window.location.reload();
```

### 3. 예산 빈 상태 테스트
```typescript
// 브라우저 콘솔에서
const budget = JSON.parse(localStorage.getItem('budget'));
budget.categories = [];
localStorage.setItem('budget', JSON.stringify(budget));
window.location.reload();
```

### 4. 검색 결과 없음 테스트
1. 지출 또는 식장 페이지 이동
2. 검색창에 존재하지 않는 키워드 입력
3. 빈 상태 확인
